Gemini投資レポートシステム開発計画書 - Gemini.md
1. システム概要
本システムは、厳選された専門的情報ソースを統合し、Gemini APIを用いて個人向け週次投資レポートを自動生成する、高機能な分析エンジンである。レポートは、マクロ経済、ミクロ経済、投資家心理、そしてオルタナティブデータの視点を統合した多層的な洞察を提供する。

情報ソース（入力）:

マクロ経済: @AC_Investor (YouTube)、Oxford Economics/Templeton Global Macro (有料レポート)

ミクロ経済/トレンド: The Information, NewsPicks, Ben Thompson (Stratechery), Noah Smith (Noahpinion), Fred Wilson (AVC)

定性情報/心理: 中嶋聡さんのメルマガ、DZHフィナンシャルリサーチ、Moomoo (空売りデータ)

客観的指標: VIX指数、主要株価指数（日経平均、S&P 500、NASDAQ）、主要暗号資産価格、為替レート（ドル円）、QUICK Data Factory (オルタナティブデータ)

レポート内容（出力）:

市場の複合的な要因分析と、各市場の個別動向。

投資家心理の多角的分析。

次なるトレンドの深掘り。

今後の展望と注視すべきポイント。

2. 実装案と技術スタック
本システムは、Pythonで開発する。

モジュール名	役割	使用技術/ライブラリ
データ収集モジュール	各情報ソースからデータを取得。有料ソースは手動入力またはスクリプトで。	Python requests, BeautifulSoup, youtube-dl, 証券会社のAPI, pandas
Gemini API連携モジュール	収集したデータをAPIに渡し、高度な分析・レポート生成を行う。	Python google-generativeai ライブラリ
レポート生成モジュール	Gemini APIの出力をMarkdown形式のレポートに整形・出力する。	Python os, markdown
スケジューリング	週に一度のレポート生成を自動化。	cron (Linux) / タスクスケジューラ (Windows)

Google スプレッドシートにエクスポート
3. 開発手順
ステップ1: 環境構築とAPIキー取得
Python 3.8+ 環境をセットアップ。

必要なライブラリをインストール: pip install google-generativeai requests beautifulsoup4 pandas youtube-dl

Gemini APIキー、および有料サービスAPIキー（利用可能な場合）を取得し、環境変数に設定する。

有料レポート（Oxford Economics, Stratecheryなど）の購読を完了する。

ステップ2: データ収集モジュール実装
各情報ソースの特性に合わせて、データ収集スクリプトを構築する。

YouTube: youtube-dl を用いて、@AC_Investorチャンネルの最新動画の文字起こしテキストを取得する。

有料レポート: 定期的にレポートが公開された際、手動でテキストをコピー＆ペーストするか、契約者ウェブサイトからの自動スクレイピング（※利用規約確認必須）を行うスクリプトを開発する。

株価・指標データ: yfinanceや証券会社のAPIを利用し、日次データを取得・保存する。

オルタナティブデータ: QUICK Data Factoryとの契約に基づき、提供されるAPIやSFTP経由でデータを自動取得する。

ステップ3: Gemini APIプロンプト設計（最も重要）
レポートの多層的な分析を可能にするため、複数の情報源を統合し、論理的推論を促す複雑なプロンプトを作成する。

プロンプトの構成例
Python

# Gemini APIへ送るプロンプトの例

prompt = """
以下の多様な情報源を統合し、グローバル市場のマクロ・ミクロ経済、投資家心理、そして次のトレンドに関する週次投資レポートを、プロフェッショナルなトーンで作成してください。レポートはMarkdown形式で、具体的な売買指示を含まず、あくまで情報提供に徹してください。

### 入力データ

- **マクロ経済（@AC_Investor & Oxford Economics）**:
  - @AC_Investor動画の文字起こし: {ac_investor_text}
  - Oxford Economicsのレポート要約（手動入力）: {oxford_economics_text}
- **トレンド/ミクロ経済（Stratechery, NewsPicks & QUICK Data）**:
  - Stratecheryニュースレターの要約（手動入力）: {stratechery_text}
  - NewsPicksの注目記事要約（手動入力）: {newspicks_text}
  - QUICK Data Factoryの最新オルタナティブデータ: {quick_data_text}
- **定性分析/投資家心理（中嶋聡, DZH & Moomoo）**:
  - 中嶋聡さんメルマガの要約（手動入力）: {nakajima_text}
  - DZHフィナンシャルリサーチのアナリスト論調（手動入力）: {dzh_text}
  - Moomooの空売り残高データ（手動入力）: {moomoo_text}
- **客観的指標**:
  - VIX指数: {vix_value}
  - 主要市場指数と為替データ: {index_data}

### レポートの構成

#### 1. 市場全体のマクロ経済分析

* **@AC_InvestorとOxford Economicsの統合分析**:
    * グローバル経済の主要トレンド（例：インフレ、金融政策）を要約。
    * 両ソースの分析を統合し、長期的な構造的変化と短期的なイベントドリブンなリスクを対比的に論じてください。

#### 2. 投資家心理の多角的分析

* **VIX指数と空売りデータの統合**:
    * VIX指数の動きから表面的な市場の恐怖度を評価。
    * Moomooの空売りデータを基に、市場の隠れた弱気センチメントと「ショートスクイーズ」のリスクを深く掘り下げてください。
* **専門家の見解**:
    * DZHフィナンシャルリサーチのアナリスト論調と中嶋聡さんの見解を比較し、プロの市場参加者の間でどのような意見の相違点や共通点があるかを考察してください。

#### 3. 次なるトレンドの深掘り

* **StratecheryとNewsPicksのクロス分析**:
    * Ben Thompsonが提唱する「フレームワーク」が、NewsPicksで話題になっている日本の特定のスタートアップや産業トレンドにどのように適用できるかを論じてください。
* **オルタナティブデータとの関連付け**:
    * QUICK Data Factoryの位置情報データやPOSデータが、特定の銘柄やセクターの動向に先行してどのような兆候を示しているかを分析してください。

#### 4. 市場別展望と結論

* **各市場の個別分析**:
    * 日本株式、米国株式、暗号資産の各市場を個別に分析し、上記で統合したマクロ・ミクロ・心理的な要因がそれぞれにどのような影響を与えているかを論じてください。
* **今週の最重要ポイントTOP3**:
    * レポート全体から、最も重要な洞察を3つに絞り、箇条書きで提示してください。
* **次週への展望**:
    * 来週の市場で特に注視すべきイベントやテーマを簡潔にまとめ、レポートを締めくくってください。
"""
ステップ4: レポート生成と出力
Gemini APIからのレスポンスをMarkdownファイルとして保存するスクリプトを作成する。report_generator.pyなどのファイル名で作成し、実行可能にする。

ステップ5: スケジューリング
cron（Linux）やタスクスケジューラ（Windows）を用いて、毎週決まった曜日・時間にreport_generator.pyが自動実行されるように設定する。
